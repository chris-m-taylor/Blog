Software and Ethics

HW1: Chapter 1Published August 27, 2020

A software should have these four traits, maintainability, dependability and security, efficiency, and acceptability. All of these traits are essential to what a professional software should possess.

1.3: What are the four important attributes that all professional software should possess? Suggest four other attributes that may sometimes be significant.

Maintainability: If you are always having to relearn code every time it needs and update or a fix, it isn’t being maintained properly. A program should be well documented to help point out what certain functions accomplish.

Dependability and Security:The software should do what it is supposed to do every time it is run without any unexpected results. If your software is unpredictable in certain situations, it is NOT dependable and needs to be fixed. Software should be difficult to steal information and difficult to inject information. If a software can be easily hacked by running a simple and KNOWN piece of malware, it is NOT secure.

Efficiency:A piece of professional software needs to be efficient. This means the software delivers what it is meant to in a reasonable amount of time. This time limit can vary depending on what exactly is being processed. A simple website page loading may take less than a second while a complex machine learning algorithm may take much longer.

Acceptability:A piece of software must be easy to understand and use for the user. It must also be compatible with other systems that are already in place.

Four other traits

A professional software may also be ethical, modern, unique, and user friendly.

Ethical: A piece of software should never be designed with malicious intent, although this is not the only ethical problem. A software may not be intentionally unethical, but as a byproduct could be unethical.

Modern:A software should be up to date with current trends and systems. This slightly ties into maintainability because it makes it possible to stay current since it is easy to update and change code.

Unique:The application should be unique and not a copy of something that already exists. That being said, it could be similar to something that exists and improve upon one or more aspects of the software.

User Friendly:This I think is one of the most important “secondary” aspects to a professional software. If your system is not user friendly, no user will want to use it. If the user struggles to learn the system the first time, that is ok. A user should NEVER have to keep relearning the system to continue to use it.

1.9: For each of the clauses in the ACM/IEEE Code of Ethics shown in Figure 1.4, propose an appropriate example that illustrates that clause.

The code of ethics are stated below with an example directly below each of them.

PUBLIC — Software engineers shall act consistently with the public interest.

A software engineer should not create a device that crashes government systems.

CLIENT AND EMPLOYER — Software engineers shall act in a manner that is in the best interests of their client and employer consistent with the public interest.

If serious safety problem occurs, talk to your employer to fix it before going directly to the client. If the employer brushes it off, it may be worth announcing publicly depending on the severity of the concern.

PRODUCT — Software engineers shall ensure that their products and related modifications meet the highest professional standards possible.

Rigorous testing should happen before the launch of the software.

JUDGMENT — Software engineers shall maintain integrity and independence in their professional judgment.

If a client employer, or coworker asks for something unethical, private information, the software engineer should use their personal judgement to deny the information and report if necessary.

MANAGEMENT — Software engineering managers and leaders shall subscribe to and promote an ethical approach to the management of software development andmaintenance.

If a security breach were to happen to users passwords. The manager should immediately tell the users there has been a breach to the system and passwords were stolen.

PROFESSION — Software engineers shall advance the integrity and reputation of the profession consistent with the public interest.

Leaders should lead by example to teach the next generation of leaders how to be ethical and continue a positive trend.

COLLEAGUES — Software engineers shall be fair to and supportive of their colleagues.

Posting answers on websites like stack overflow can help to build up fellow software engineers and help the community.

SELF — Software engineers shall participate in lifelong learning regarding the practice of their profession and shall promote an ethical approach to the practice of the profession.

Once out of school, a software engineer should always be learning, whether that be getting certifications or taking Udemy courses.
