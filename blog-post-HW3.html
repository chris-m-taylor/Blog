Availability, Reliability, and Safety

HW3: Chapters 11 & 12Published September 4th, 2020

11.4 What is the common characteristic of all architectural styles that are geared to supporting software fault tolerance?

They either deal with availability or reliability problems which mean they need a high amount of redundancy and diversity. A system that is supporting fault tolerance needs to have multiple systems that output the same thing. These multiple systems, however, need to be programmed in different ways, and/or use different hardware in each system. This is to allow the output to be checked with different systems that should produce the same output.

A good analogy would be if you had a math problem that you didn’t know how to solve. You can either take it to one of your friends and have him solve the same problem 3 times and compare his three answers. Or you can take it to three different friends, and if 2 of them have the same answer, you can assume that the answer is correct.

This is the importance of not only have redundancy in your system, but also diversity.

11.7 It has been suggested that the control software for a radiation therapy machine, used to treat patients with cancer, should be implemented using N-version programming. Comment on whether or not you think this is a good suggestion.

First, let’s define what N-version programming is. N-version programming is a type of system that has multiple self-monitoring programs, or multiple pieces of hardware with the same or different program, running at the same time. If an input is passed in, it would be sent through the three different programs/hardware and then the output would be compared.

This level of fault tolerance is more expensive than just having one system and solving errors as they arise. Since the system will be dealing with radiation therapy, the machine must have a high level of reliability. N-version programming would be worth it in this case since the stakes are much higher than a system that does not involve the lives of humans.

11.9 Explain why you should explicitly handle all exceptions in a system that is intended to have a high level of availability.

Explicitly handling all exceptions will allow the system to continue running even if an error occurs. Instead of the error stopping the program completely, we can catch the error at a lower level and handle it, so it does not get passed further along the chain.

Unfortunately, the more exceptions that are account for, the more expensive the software becomes. There is an exponential curve where the cost per error detected rises as the number of errors remaining decreases.

Now, if the budget is there, the more exceptions you are able to process will greatly reduce the chances of the system crashing or becoming unavailable.

12.5 A train protection system automatically applies the brakes of a train if the speed limit for a segment of track is exceeded, or if the train enters a track segment that is currently signaled with a red light (i.e., the segment should not be entered). There are two critical-safety requirements for this train protection system:

The train shall not enter a segment of track that is signaled with a red light.
The train shall not exceed the specified speed limit for a section of track.
Assuming that the signal status and the speed limit for the track segment are transmitted to on-board software on the train before it enters the track segment, propose five possible functional system requirements for the onboard software that may be generated from the system safety requirements.

It is important to have a good hazard detection and removal system in place when dealing with situations and can cause injury or death. Five possible functional system requirements for the system above are as follows:

  • The system will measure the speed limit the train is going as well as the speed limit set for the track it is on.
  • The system shall deliver an alarm to the train operator BEFORE the speed limit is exceeded.
  • The system will not disengage the brakes if they were activated by reaching one of the above potential risks
  • The system will measure the distance between the front of the train and any red-light zones it might approach.
  • The system shall deliver an alarm to the train operator if they are approaching a track that is currently signals with a red light.
